# Mantis Jumping Analysis

This repository contains analysis for mantis jumping and peering experiments using DeepLabCut-derived body-part coordinate data. The notebook converts raw coordinates into body center-of-mass trajectories, computes takeoff kinematics, and plots relationships between target distance/angle and jump performance.

## Overview

The analysis includes:

- center-of-mass (COM) estimation from labeled body parts
- Butterworth filtering of motion trajectories
- frame-by-frame speed and direction calculations
- takeoff speed and takeoff angle computation
- target distance and target angle estimation
- peering frequency and amplitude analysis
- figure generation for manuscript-style results

## Repository contents

- `Code.ipynb` – main Jupyter notebook with the full analysis pipeline and figure-generation code.
- `JumpKinematics.csv` – summary table of jump kinematics and COM-derived variables.
- `PeeringKinematics.csv` – summary table of peering behavior and oscillation metrics.
- `Statistics_Summary.csv` – aggregated model and statistics output generated by the notebook.
- `CSV files with position coordinates/` – folder containing raw trial files with labeled body-part coordinates.

## Required data files

The notebook references the following CSV files:

- `JumpKinematics.csv` – jump trial metadata plus kinematic results.
- `PeeringKinematics.csv` – peering trial metadata plus frequency/amplitude measurements.
- `Statistics_Summary.csv` – overall statistics output.
- `CSV files with position coordinates/` – raw input trial files used for both jump and peering computations.

Note: `Mantis distance and angle.csv` is referenced in many analysis cells but is not included in the current repository snapshot. If you want to reproduce the figure-generation workflow, run the metadata calculation sections in `Code.ipynb` or provide that file separately.

## Expected input structure

Raw trial CSV files should include body-part coordinate columns and metadata fields such as:

- `filename`
- `takeofftime(ms)`
- `touchdown(ms)`
- `pixelsize(m)`
- `framerate(Hz)`
- `TargetPlatform_x`
- `TargetPlatform_y`
- body-part columns like `Head_x`, `Head_y`, `ProThorax_x`, `ProThorax_y`, `MesoThorax_x`, `MesoThorax_y`, etc.

The jump metadata table also contains mass and body-part mass values used for COM estimation.

## Usage

1. Open `Code.ipynb` in Jupyter Notebook or JupyterLab.
2. Run the notebook cells from top to bottom.
3. If `Mantis distance and angle.csv` is missing, run the metadata computation section to generate the required dataset.
4. Inspect generated figures, summary tables, and saved CSV outputs.
